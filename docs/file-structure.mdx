# File Structure Guide

## Overview

Understanding where to put your files is crucial. This guide shows you exactly where everything goes.

---

## Root Folder Structure

Your website has this basic structure:

```
your-website/
├── settings/
│   ├── config.jsonc
│   ├── pages.jsonc
│   ├── projects.jsonc
│   └── lang.jsonc
├── assets/
│   ├── media/
│   │   ├── img/
│   │   │   ├── main/
│   │   │   ├── home/
│   │   │   ├── project-categories/
│   │   │   ├── projects/
│   │   │   └── others/
│   │   └── fonts/
│   └── pages/
│       ├── markdown/
│       │   ├── projects/
│       │   └── others/
│       └── pdf/
│           └── projects/
├── public/
└── src/
```

---

## Settings Files

### Where: `/settings/`

This folder contains all your configuration files:

| File | Purpose |
|------|---------|
| `config.jsonc` | Website appearance, colors, logo, SEO |
| `pages.jsonc` | Categories (Projects) and pages (About, Contact) |
| `projects.jsonc` | Your actual work samples |
| `lang.jsonc` | Text labels and translations |

## Content Files

### Where: `/assets/pages/`

#### Markdown Files (For Markdown & Page View Projects)

```
assets/pages/
└── markdown/
    ├── projects/
    │   ├── appearance1/
    │   │   └── example.md
    │   └── [your-project-name]/
    │       └── content.md
    └── others/
        ├── about.md
        └── [your-page-name].md
```

**Markdown File Example:**
```markdown
# Project Title

## Overview
This is the project description...

## Details
- Year: 2024
- Location: New York
- Area: 500 sqm

## Images
![Project Image](/assets/media/img/projects/your-project/1.webp)

## Conclusion
This project showcases...
```

## URL Path Reference

### For Pages.jsonc

| Page Type | URL Field Example |
|-----------|------------------|
| Home | (default) |
| Contact | "/contact" |
| Projects | "/projects" |
| Category | "/appearances" |
| Others Page | "/about" |

### For Projects.jsonc

| Project | URL Field Example |
|---------|------------------|
| Project in category | "/page-view" (becomes: /projects/appearances/page-view) |

### Full URL Structure

```
Home:           yoursite.com/
Contact:        yoursite.com/contact
Projects List:  yoursite.com/projects
Category:       yoursite.com/projects/appearances
  Project:      yoursite.com/projects/appearances/page-view
About:          yoursite.com/about
Services:       yoursite.com/services
```

## Common Mistakes to Avoid

### Mistake 1: Wrong Image Path
```json
// ❌ Wrong
"img": "/assets/media/projects/image.webp"

// ✅ Correct
"img": "/assets/media/img/projects/project-name/image.webp"
```

### Mistake 2: Forgetting /img/ in Path
```json
// ❌ Wrong
"img": "/assets/media/project-categories/..."

// ✅ Correct  
"img": "/assets/media/img/project-categories/..."
```

### Mistake 3: Wrong Category Key
```json
// pages.jsonc has: "residential": { "base": { "projectKey": "residential" } }

// ❌ Wrong - key doesn't match
"parentKey": "category1"

// ✅ Correct - matches pages.jsonc
"parentKey": "residential"
```

### Mistake 4: Spaces in Folder Names
```json
// ❌ Wrong
"img": "/assets/media/img/projects/My Project/image.webp"

// ✅ Correct - use hyphens or underscores
"img": "/assets/media/img/projects/my-project/image.webp"
```

## Creating Folders: Step by Step

### Step 1: For a New Category

1. Create folder: `assets/media/img/project-categories/your-category/`
2. Upload banner image: `banner.webp`
3. (Optional) Add SEO images: `banner-og.webp`, `banner-tc.webp`

### Step 2: For a New Project

1. Create folder: `assets/media/img/projects/your-project/`
2. Upload images:
   - `banner.webp` (cover)
   - `banner-og.webp` (social)
   - `banner-tc.webp` (twitter)
   - `1.webp`, `2.webp` (slider images)

### Step 3: For Markdown Content

1. Create folder: `assets/pages/markdown/projects/your-project/`
2. Create file: `content.md`
3. Write your content

## File Format Recommendations

### Use WebP Format
Always prefer `.webp` format for images:
- ✅ `.webp` - Best quality, smallest file size
- ✅ `.png` - Good for transparency
- ✅ `.jpg` - Acceptable for photos
- ❌ `.gif` - Only for animations

### Image Optimization Tips
1. Compress images before uploading
2. Use appropriate sizes (not too large)
3. Use consistent aspect ratios for similar images

## Summary

Key takeaways:

1. **Images go in** `/assets/media/img/`
2. **Content goes in** `/assets/pages/`
3. **Settings go in** `/settings/`
4. **Always use correct paths** - check twice!
5. **Use WebP format** for best performance
