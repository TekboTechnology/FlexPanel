# Projects.jsonc - Complete Guide

## What is projects.jsonc?

This file contains **your actual work samples**. While `pages.jsonc` creates the categories (folders), `projects.jsonc` fills them with content (files).

Think of it like:
- `pages.jsonc` = Empty folders labeled "Residential", "Commercial"
- `projects.jsonc` = Actual project files inside those folders

**File Location:** `settings/projects.jsonc`

---

## Project Structure

Each project has several parts:

```json
"project1": {
  "parentKey": "category1",
  "base": { ... },
  "opengraph": { ... },
  "twitterCard": { ... },
  "pageIn": { ... }
}
```

---

## Step-by-Step: Creating a New Project

### Step 1: Define Parent Category

```json
"myProject": {
  "parentKey": "category1"
}
```

**Important:** The `parentKey` must match a category from `pages.jsonc`!

| If your category in pages.jsonc is... | Use this parentKey in projects.jsonc |
|--------------------------------------|-------------------------------------|
| `"category1"` | `"category1"` |
| `"residential"` | `"residential"` |
| `"commercial"` | `"commercial"` |

---

### Step 2: Basic Information

```json
"base": {
  "title": "Modern Villa Design",
  "img": "/assets/media/img/projects/my-project/banner.webp",
  "desc": "A beautiful modern villa with minimalist design",
  "keywords": "villa, modern, architecture, residential",
  "url": "/modern-villa"
}
```

| Field | What It Does | Example |
|-------|--------------|---------|
| `title` | Project name | "Modern Villa" |
| `img` | Cover image (1500x500) | "/assets/.../banner.webp" |
| `desc` | Short description for lists | "Beautiful modern home" |
| `keywords` | SEO keywords | "villa, modern, home" |
| `url` | URL path | "/modern-villa" |

---

### Step 4: Content Type (The Most Important Part!)

This defines **how your project page looks**. You have 3 options:

#### Option A: Page View (Inline Content)

Best for: Adding images and text directly in JSON

```json
"pageIn": {
  "settings": {
    "select": "page"
  },
  "data": {
    "page": {
      "title": "Modern Villa Design",
      "desc": "Full description of the project...",
      "settings": {
        "returnCategoryEnable": true
      },
      "images": {
        "settings": {
          "autoplay": true,
          "autoplaySpeed": 3000,
          "animSpeed": 500,
          "arrows": true,
          "infinite": true
        },
        "data": [
          "/assets/media/img/projects/my-project/1.webp",
          "/assets/media/img/projects/my-project/2.webp",
          "/assets/media/img/projects/my-project/3.webp"
        ]
      }
    }
  }
}
```

#### Option B: Markdown View

Best for: Long-form content, documentation, blog posts

```json
"pageIn": {
  "settings": {
    "select": "mdpatch"
  },
  "data": {
    "mdpatch": "/assets/pages/markdown/projects/my-project.md"
  }
}
```

**How to create markdown content:**
1. Create folder: `/assets/pages/markdown/projects/`
2. Create file: `my-project.md`
3. Write content using Markdown format

**Markdown Example:**
```markdown
# Modern Villa Project

## Overview
This is a beautiful modern villa located in...

## Details
- Location: New York
- Year: 2024
- Area: 500 sqm

## Gallery
![Villa Exterior](/assets/media/img/projects/my-project/1.webp)

## Conclusion
This project showcases...
```

#### Option C: PDF View

Best for: Brochures, catalogs, formal documents

```json
"pageIn": {
  "settings": {
    "select": "pdf"
  },
  "data": {
    "pdf": {
      "dark": "/assets/pages/pdf/projects/my-project_dark.pdf",
      "light": "/assets/pages/pdf/projects/my-project_light.pdf"
    }
  }
}
```

**PDF Requirements:**
- One PDF for dark mode
- One PDF for light mode
- Recommended size: A4 or Letter

## Image Slider Settings

When using "page" content type, you can customize the image slider:

```json
"images": {
  "settings": {
    "autoplay": true,        // Auto-slide images (true/false)
    "autoplaySpeed": 3000,   // Time between slides (milliseconds)
    "animSpeed": 500,        // Transition speed
    "arrows": true,          // Show navigation arrows
    "infinite": true         // Loop continuously
  },
  "data": [
    "/assets/.../image1.webp",
    "/assets/.../image2.webp",
    "/assets/.../image3.webp"
  ]
}
```

| Setting | What It Does |
|---------|--------------|
| `autoplaySpeed` | 1000 = 1 second, 3000 = 3 seconds |
| `animSpeed` | Lower = faster animation |
| `arrows` | Show < > buttons on images |
| `infinite` | When reaching last image, go to first |

## File Structure

Your project images should be organized like this:

```
/assets/
  media/
    img/
      projects/
        project-name/
          banner.webp          (1500x500 - cover image)
          banner-og.webp       (1200x630 - social share)
          banner-tc.webp       (1200x628 - Twitter share)
          1.webp               (1920x1080 - slider image 1)
          2.webp               (1920x1080 - slider image 2)
          3.webp               (1920x1080 - slider image 3)

/assets/
  pages/
    markdown/
      projects/
        project-name.md        (for markdown view)

    pdf/
      projects/
        project-name_dark.pdf  (for PDF dark mode)
        project-name_light.pdf (for PDF light mode)
```

## Important Rules

### Rule 1: Match Parent Category
The `parentKey` MUST match a category in `pages.jsonc`:
```json
// pages.jsonc has: "residential": { "base": { "projectKey": "residential" } }
// projects.jsonc must use: "parentKey": "residential"
```

### Rule 2: Unique Project Keys
Each project needs a unique key at the top:
```json
"project1": { ... },
"project2": { ... },
"myUniqueProjectName": { ... }
```

### Rule 3: Unique URLs
Each project URL must be unique:
```json
"url": "/modern-villa"     // ✓ Good
"url": "/modern-villa"     // ✗ Bad - duplicate!
```
